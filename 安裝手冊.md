# 安裝手冊 — 在自己的電腦上跑這套 Java 教材

這份手冊帶你**從零**把環境裝好、把本專案打開、跑出第一支程式。
給第一次接觸的同學照著做即可;Windows / macOS / Linux 都適用。

> **你只需要選一個 IDE**:Eclipse **或** VS Code(下面分成兩條路線,挑一條走完即可)。
> 兩邊讀的是**同一份專案檔**,將來想換另一個也不必重設定。
> 想知道更多(課程地圖、環境細節、學生留白清單),看 [README.md](README.md)。

---

## 0. 總覽:你要裝哪些東西

| 要裝的 | 說明 | 兩條路線都要? |
|---|---|---|
| **JDK(Java)** | 編譯與執行 Java 的核心,**建議 JDK 21(LTS)** | ✅ 都要 |
| **IDE** | Eclipse **或** VS Code,二選一 | 選一個 |
| **本專案檔** | 這個 repo(老師會給你取得方式) | ✅ 都要 |

順序:**先裝 JDK(第 1 步)→ 再裝你選的 IDE → 取得專案 → 打開執行**。

## 1. 安裝 JDK(兩條路線都需要,先做)

1. 到 **<https://adoptium.net/>**(Eclipse Temurin,免費的 OpenJDK)下載 **JDK 21 (LTS)**。
   - 選你的作業系統的 **`.msi`(Windows)/ `.pkg`(macOS)** 安裝版。
2. 安裝時,若安裝精靈有選項,請勾選 **「Add to PATH」** 與 **「Set JAVA_HOME」**。
3. 驗證:打開終端機(Windows 用 PowerShell 或 cmd),輸入:
   ```
   java -version
   ```
   有印出版本(例如 `openjdk version "21.0.x"`)就成功。

> **為什麼是 21?** 本教材的語言層級全部鎖在 **Java 8**(所以你裝哪個新版都**不影響教材行為**),
> 但 VS Code 的 Java 語言伺服器**需要 JDK 17 以上**才跑得起來,Eclipse 也建議自備一顆現代 JDK。
> 裝 21(LTS)兩條路線都通;裝 25(最新)也可以,已實測編譯全數通過。

## 3A. 路線一:用 Eclipse

### 3A-1　安裝 Eclipse
到 **<https://www.eclipse.org/downloads/packages/>** 下載 **「Eclipse IDE for Java Developers」**,安裝後開啟,
第一次會問 workspace 位置(隨便選一個資料夾即可,這只是 Eclipse 的工作區,**不是**專案位置)。

### 3A-2　匯入專案
1. `File → Import…`
2. 選 `General → Existing Projects into Workspace` → `Next`
3. **Root directory** 選你第 2 步取得的 **repo 根目錄**
4. 下方會列出所有 Unit 專案,勾你要用的(每個 Unit 是獨立專案,可只匯入需要的)
5. **不要勾** ☐ *Copy projects into workspace*(直接在原位置使用)
6. `Finish`

> 各專案的 `.classpath`、編碼、Karel 的 jar 都已經在 git 裡設好,**匯入後直接可用,不必再設定任何東西**。

### 3A-3　(選用)指定要用哪顆 JDK
`Window → Preferences → Java → Installed JREs`,把你第 1 步裝的 JDK 勾起來。
不指定也行——Eclipse 會用內建 JRE,因為教材鎖 1.8,行為一樣。

### 3A-4　跑第一支程式(驗收)
1. 展開 `SetupFiles` → `src` → 開 **`CheckEnv.java`**
2. 右鍵 → `Run As → Java Application`
3. 對照第 5 節的預期輸出(中文那行不能亂碼)

---

## 3B. 路線二:用 VS Code

### 3B-1　安裝 VS Code + Java 擴充套件
1. 到 **<https://code.visualstudio.com/>** 下載安裝 VS Code。
2. 安裝 **「Extension Pack for Java」**(微軟出的 Java 全家桶):
   左側 Extensions(`Ctrl+Shift+X`)搜尋安裝;或第一次打開 `.java` 檔時右下角會自動建議(本專案已把建議寫在 `.vscode/extensions.json`),按下去裝即可。
3. 確認第 1 步的 JDK 已裝好(VS Code 的 Java 功能**需要 JDK 17+**)。

### 3B-2　打開專案(**兩種開法,擇一**)
- **開單一 Unit(最單純、保證正確)**:`File → Open Folder…` 直接開某個 Unit 資料夾(例如 `Unit1`)。
- **一次看整包 repo**:`File → Open Workspace from File…` → 選 repo 根目錄的 **`fcps-jkarel-units.code-workspace`**。
  它把每個 Unit 列為獨立資料夾,執行時工作目錄會正確落在各 Unit 根,且 Source Control 面板看得到整包、可直接 commit/push。

> ⚠️ **不要**用 `Open Folder…` 開「**repo 根目錄**」然後去跑 Karel 程式——那樣工作目錄會是 repo 根、
> 而非 Unit 根,Karel 的 `maps/xxx.map` 會找不到。要嘛開單一 Unit 資料夾,要嘛用上面的 `.code-workspace`。

### 3B-3　(選用)指定要用哪顆 JDK
教材鎖 1.8,VS Code 用哪顆 JDK 跑都不影響行為。要明確指定或在多顆之間切換,
在你自己的 **User Settings**(不是專案設定,因為路徑每台機器不同)加 `java.configuration.runtimes`——
完整範例見 [README.md 的「指定 JDK」一節](README.md)。

### 3B-4　跑第一支程式(驗收)
1. 開 `SetupFiles/src/CheckEnv.java`
2. 按檔案上方 `main` 方法旁出現的 **`▶ Run`** CodeLens
3. 對照第 5 節的預期輸出

---

## 4. 跑一支 Karel 確認圖形也正常(選用,但建議)

開 `Unit1`,找一支含 `main` 的 Driver 執行,
應該會跳出 Karel 機器人的視窗、讀得到地圖。**只有 Unit1 會讀 `maps/`**,
其餘 Unit 沒這顧慮。若 Karel 視窗出不來或地圖讀不到,通常是「工作目錄不是專案根」——
回到 3B-2 的開法規則即可。

## 5. CheckEnv 應該印出什麼

```
java.version  = 21.0.x          ← 你裝的 JDK 版本(8 / 17 / 21 / 25 都算正常)
java.vendor   = Eclipse Adoptium
java.home     = C:\Program Files\Eclipse Adoptium\jdk-21...   ← 實際在用的 JDK 路徑
file.encoding = UTF-8           ← 必須是 UTF-8
user.dir      = ...\SetupFiles  ← 執行時的工作目錄
os.name       = Windows 11
中文測試:如果這行是亂碼,代表 console 編碼有問題    ← 這行要看得懂、不能是亂碼
```

- `file.encoding = UTF-8` 且**最後一行中文正常** → 環境 OK。
- 設定畫面說的是「**應該**用哪顆 JDK」,CheckEnv 印的是「**正在**用哪顆」——到任何新機器,**第一件事就是跑它**。

## 6. 常見狀況

| 症狀 | 多半的原因 / 解法 |
|---|---|
| **學生版(UnitN)有些 Lab 顯示紅字/錯誤** | **正常**——那些 Lab 是刻意留白給學生寫的。完整清單見 [README.md 的「學生版的紅字是正常的」](README.md)。 |
| **Karel 視窗出不來 / `maps/xxx.map` 找不到** | 工作目錄不是專案根。Eclipse 預設就是專案根;VS Code 請照 3B-2 的開法(別開 repo 根)。 |
| **中文變亂碼** | 教材檔案已鎖 UTF-8;若 CheckEnv 最後一行亂碼,多半是 Windows console 顯示問題,程式本身沒事。 |
| **VS Code 一直顯示舊狀態 / 紅字清不掉** | 指令面板(`Ctrl+Shift+P`)執行 `Java: Clean Java Language Server Workspace` 重建。 |
| **Eclipse pull 大更新後一堆紅字** | 全選專案 → `F5` 重新整理 → `Project → Clean…`。詳見 [README 的「換新機器 SOP」](README.md)。 |
